# 获客工作流技能

端到端获客流程定义，包括技能调度、质量检查、结果输出。

---

## 流程总览

```
Phase 1: 搜索 (10分钟)
    │
    ▼
Phase 2: 背调 (30分钟)
    │
    ▼
Phase 3: 筛选 (5分钟)
    │
    ▼
Phase 4: 触达 (10分钟)
```

## 阶段详解

### Phase 1: 搜索
**目标**: 发现15-30家潜在客户，并获取有效联系方式

**技能选择**:
| 场景 | 调用技能 |
|------|----------|
| LinkedIn搜索 | linkedin |
| 海关数据（特易） | teyi-customs |
| Google搜索 | multi-search-engine |
| Facebook挖掘 | scrapling |

**获取联系方式（Phase 1.5，必须执行）**:

> 搜索完成后，必须获取每个客户的真实联系方式，不能只靠猜测邮箱。

| 方法 | 适用场景 | 操作 |
|------|----------|------|
| 特易详情页「联系方式」 | 通过特易搜索的客户 | 点击公司名 → 左侧「商业数据」→「联系方式」 |
| 易搜邮（EaseSearch Mail） | 详情页无邮箱 / 需要决策人邮箱 | 特易平台 → 易搜邮 → 输入公司名/网站 |
| LinkedIn直接查看 | LinkedIn搜索的客户 | 查看 Contact Info 或 About 页面 |
| 官网找页面 | 通用搜索的客户 | 访问官网 Contact/About 页面 |

**质量检查点**:
- [ ] 搜索关键词是否准确？
- [ ] 结果是否属于目标行业？
- [ ] **每个客户是否有有效邮箱？（必须）**
- [ ] 联系方式来源是否可靠（详情页/易搜邮 > 网页猜测）？

**联系方式质量标准**:
- ✅ 优先：决策人/采购经理邮箱（S级）
- ✅ 可用：公司通用邮箱 info@ / sales@（A级）
- ⚠️ 低质：仅网站表单（B级，需额外核实）
- ❌ 不可用：无任何联系方式 → 用易搜邮补充后再继续

**输出**: 公司列表 (JSON格式)，每条记录必须包含 email 字段

### Phase 2: 背调
**目标**: 评估客户质量
**技能选择**:
| 场景 | 调用技能 |
|------|----------|
| 快速背调 | company-research |
| 深度背调 | in-depth-research |
| 市场分析 | market-research |
| 竞品识别 | honglong-products |

**质量检查点**:
- [ ] 评分是否合理？
- [ ] 是否识别出竞争对手？
- [ ] 关键信息是否完整？

**输出**: 评分报告 (A/B/C/D级)

### Phase 3: 筛选
**目标**: 确定高价值客户
**筛选条件**:
- 评分 ≥ 75分
- 非竞争对手
- 有联系方式

**输出**: 高价值客户列表 (2-5家)

### Phase 4: 触达
**目标**: 发送带产品资料的开发信，并启动跟进序列

**技能选择**:
| 场景 | 调用技能 |
|------|----------|
| NAS挂载 | nas-file-reader |
| 获取产品资料 | nas-file-reader + honglong-products |
| 生成开发信 | email-marketing |
| 配置邮箱 | email-sender |
| 发送邮件 | email-sender |
| 控制发送节奏 | delivery-queue |

**📋 开发信签名规范**:

> 开发信签名使用**业务员本人的联系方式**，不使用公司公共邮箱。

签名应包含：
- 业务员姓名
- 公司名称
- 个人手机/WhatsApp
- 个人工作邮箱
- 公司官网
- 公司地址

**邮件发送配置**:

> 每位业务员需自行配置发件邮箱（首次使用运行 `email-sender` 技能的 setup 向导）。

配置项：
- SMTP 服务器 + 端口 + 加密方式
- 发件邮箱账号 + 授权码
- 发件人名称

**发送频率限制**（通用规范）：
- 每天发送上限：< 200 封
- 每批间隔：2-5 分钟（模拟真人节奏，避免被标记垃圾邮件）
- 单日新客户：建议 ≤ 30 封

**前置检查**:
- [ ] 开发信签名是否使用本人联系方式？
- [ ] 开发信是否通过质量评分 ≥ 7.0？
- [ ] 邮箱是否已配置？（未配置先运行 email-sender setup）
- [ ] NAS是否已挂载（产品资料）？

**详细步骤**:
1. **生成开发信** - 个性化 + 针对目标客户痛点
2. **开发信润色** - humanize-ai-text 评分 ≥ 7.0
3. **配置邮箱** - 163 SMTP（首次需要运行setup）
4. **发送 D0 首封邮件** - 附产品资料链接
5. **设置跟进计划** - D3/D5/D14 三轮自动跟进

**⭐ 四轮跟进节奏（Drip Campaign）**:

| 轮次 | 时间 | 内容 | 技能 |
|------|------|------|------|
| D0 | 发送当天 | 首封开发信 + 产品介绍 | email-sender |
| D3 | 第3天 09:00 | LinkedIn 连接请求 + 简短留言 | linkedin |
| D5 | 第5天 09:00 | 跟进邮件：强调价值主张 + 案例 | follow_up_emails.py |
| D14 | 第14天 09:00 | 最终跟进：提供 Free Sample / Demo | follow_up_emails.py |

**跟进邮件脚本**（已就绪）：
```bash
# D5 跟进
python follow_up_emails.py day5

# D14 最终跟进
python follow_up_emails.py day14
```

**质量检查点**:
- [ ] 开发信签名是否使用本人联系方式（非公司公共邮箱）？
- [ ] 开发信是否个性化（提到客户公司名/产品/市场痛点）？
- [ ] 发送是否成功（检查发送日志）？
- [ ] D3/D5/D14 自动化任务是否已创建？

**输出**: 发送记录 + 四轮跟进计划 + 产品资料链接
