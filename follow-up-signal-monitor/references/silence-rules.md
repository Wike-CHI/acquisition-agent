# 沉默检测规则 v1.0.0

> 定义沉默天数计算逻辑、触发条件、与CRM的联动规则。

---

## 沉默天数计算规则

### 基准时间点

以最后一次有效触达为起点：

| 触达类型 | 基准时间 |
|---------|---------|
| 发出提案 | 提案生成时间 |
| 发出开发信 | 邮件发送时间 |
| 发出报价单 | 报价单生成时间 |
| 收到客户回复 | 重置为0，开始新一轮计时 |
| 手动标记「暂停」 | 暂停计时 |

### 有效触达定义

```
✅ 有效触达：
- 提案已发送（holo-proposal-generator）
- 开发信已发送（cold-email-generator → email-sender）
- WhatsApp消息已发送（whatsapp-outreach）
- 报价单已发送（smart-quote）

❌ 无效触达：
- 草稿未发送
- 邮件被退回（Bounce）
- 消息发送失败
```

### 天数计算公式

```
沉默天数 = 当前日期 - 最后有效触达日期

示例：
- 提案发出：2026-04-09
- 今天：2026-04-21
- 沉默天数：12天 → 触发 A2 案例推送
```

## 沉默分级规则

### 等级定义

```
NORMAL（正常等待）: 0-3天
├─ 客户可能在内部审批流程中
├─ 等待是正常的
└─ 无需任何动作

ATTENTION（轻度关注）: 4-6天
├─ 开始关注邮件打开状态
├─ 有打开信号 → 观察
├─ 无打开信号 → 标记关注
└─ 业务员可以选择性发送简短确认

WARNING（警戒）: 7-13天  ★重点触发
├─ 正常沉默上限，必须触发跟进
├─ 自动生成价值型跟进草稿
├─ 等待业务员确认后发送
└─ 避免重复发送原方案内容

DEEP（深度跟进）: 14-29天  ★深度触发
├─ 客户可能需要更多信息
├─ 触发深度价值推送
├─ 可以尝试电话/WhatsApp触达
└─ 记录客户无响应的可能原因

WEAK（弱信号）: 30天+
├─ 客户暂无需求，降低触达频率
├─ 标记弱信号到CRM
├─ 移入季度更新名单
└─ 6个月后再自然复联
```

## 触发动作规则

### 触发频率控制

```
同一个客户：
- 7天内只触发一次跟进（不论手动/自动）
- 价值型跟进内容不能连续两次相同类型
- 最大跟进频率：每周一次
```

### 触达渠道优先级

```
根据沉默等级选择触达渠道：

WARNING (7-13天)：
  主渠道：邮件（价值型内容）
  备渠道：WhatsApp（如果有号码）

DEEP (14-29天)：
  主渠道：邮件 + WhatsApp
  可选：建议业务员电话

WEAK (30天+)：
  停止主动触达
  仅做邮件季度更新（每季度一次）
```

### 发送前检查

```
每次发送前，系统自动检查：

1. 邮件地址是否有效（未退回）
   ❌ 邮件被退回 → 跳过邮件，更换WhatsApp

2. 跟进内容是否与上次不同
   ❌ 与上次内容相同 → 更换模板类型

3. 跟进频率是否合规（7天内只发一次）
   ❌ 7天内已发过 → 跳过，等待下一个窗口

4. 客户状态是否允许触达
   ❌ 客户已标记「暂停」→ 不触达
   ❌ 客户已成交 → 移入售后状态
   ❌ 客户已明确拒绝 → 移入丢单状态
```

## CRM状态联动规则

### 客户状态机

```
                    ┌─────────────┐
                    │   新客户     │
                    └──────┬──────┘
                           │ 首次有效触达
                    ┌──────▼──────┐
              ┌───►│  进行中     │◄─────────┐
              │     │ （待跟进）   │           │
              │     └──────┬──────┘           │
              │            │                 │
              │     沉默天数≥7天              │ 客户回复
              │            │                 │
              │     ┌──────▼──────┐          │
              │     │   警戒      │──────────┘
              │     │  （待触发）  │
              │     └──────┬──────┘
              │            │ 发送价值型跟进
              │     ┌──────▼──────┐
              │     │ 跟进已发送   │
              │     │ （等待回应） │
              │     └──────┬──────┘
              │            │
    沉默≥30天 │            │ 客户回复
              │     ┌──────▼──────┐
              │     │ 意向确认    │
              │     └──────┬──────┘
              │            │
              │     ┌──────▼──────┐
              └──► │   成交      │
                    └─────────────┘

        客户明确拒绝 ┌─────────────┐
              ──────►│   丢单      │
                    └─────────────┘
```

### CRM字段更新

```
每次状态变化，自动更新CRM字段：

触发WARNING时：
  status: "进行中" → "警戒"
  last_followup_date: 今天
  followup_tier: "warning"
  followup_count: +1
  next_action: "发送价值型跟进"

发送跟进后：
  status: "警戒" → "跟进已发送"
  last_email_sent: 今天
  email_subject: 跟进邮件主题

客户回复时：
  status: → "意向确认"
  silence_days: 0
  last_contact_date: 今天

沉默≥30天时：
  status: → "弱信号"
  signal_strength: "weak"
  next_contact_date: +90天（季度更新）
```

## 特殊客户规则

### 新客户 vs 老客户

```
新客户（首次触达30天内）：
  - 警戒期缩短为5天
  - 优先使用 A3（付款优化）触发跟进
  - 更aggressive的电话跟进

老客户（已有历史交易）：
  - 正常7天警戒
  - 优先使用 A2（案例匹配）
  - 可以提及历史合作

流失预警客户（长期未复购）：
  - 沉默14天即触发B3（软着陆）
  - 优先发送新产品/更新信息
```

### 大客户规则

```
ICP评分 ≥ 85分的大客户：
  - 警戒期缩短为5天
  - 优先使用 WhatsApp + 邮件双触达
  - 建议业务员主动电话
  - B2（有效期提醒）提前到10天发出
```

## 邮件已读回执检测

### 网易163已读回执

163邮箱支持「已读回执请求」，但依赖对方邮件客户端配合。

检测逻辑：

```
发出邮件时：
  记录发送时间 + 邮件ID

定期检查（每小时）：
  尝试获取邮件送达状态（IMAP的FLAGS）

判断逻辑：
  - 有新邮件从客户地址来 → 客户看了我们的邮件（概率高）
  - 没有新邮件，但过了48小时 → 客户可能没看到 → 升一级触发

注意：
  网易163没有标准的「已读回执」API
  只能通过「客户是否回复」来间接判断
  这是B2B邮件营销的普遍限制
```

## 配置参数

```python
# 沉默天数阈值（可调整）
SILENCE_THRESHOLDS = {
    'attention': 4,    # 轻度关注
    'warning': 7,      # 警戒（必须触发跟进）
    'deep': 14,        # 深度跟进
    'weak': 30,        # 弱信号
}

# 跟进频率控制
FOLLOWUP_COOLDOWN_DAYS = 7       # 同客户最小跟进间隔
MAX_FOLLOWUP_PER_WEEK = 1        # 每周最多跟进一次

# 弱信号处理
WEAK_SIGNAL_RESURFACE_DAYS = 90  # 弱信号客户多久后自然复联
WEAK_SIGNAL_EMAIL_FREQ = 90      # 季度更新邮件频率（天）
```
