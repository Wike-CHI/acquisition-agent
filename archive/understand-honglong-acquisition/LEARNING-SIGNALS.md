# 学习信号触发机制

> AI自动识别学习机会，持续优化获客能力

---

## 5种核心学习信号

### 1. 修正信号（Correction）

**触发词**：
- "不对"、"错了"、"不是这样的"
- "应该"、"实际上是"
- "我之前说过"、"记住"
- "别再这样"、"停止X"

**记录位置**：`~/self-improving/corrections.md`

**记录格式**：
```markdown
## [日期] 修正：[简短描述]
**场景**：[任务类型]
**错误**：[AI做了什么]
**正确**：[应该做什么]
**来源**：[用户原话]
```

**示例**：
```markdown
## 2026-04-02 修正：LinkedIn决策人搜索工具
**场景**：海外客户决策人查找
**错误**：使用linkedin MCP工具
**正确**：使用Exa搜索 site:linkedin.com
**来源**："不对，LinkedIn MCP有速率限制，应该用Exa"
```

---

### 2. 偏好信号（Preference）

**触发词**：
- "我喜欢"、"我偏好"
- "总是这样"、"永远这样"
- "我的风格是"
- "对于X项目，用Y"

**记录位置**：`~/self-improving/memory.md` (HOT层)

**记录格式**：
```markdown
## 偏好：[主题]
**规则**：[具体偏好]
**适用范围**：[项目/场景]
```

**示例**：
```markdown
## 偏好：开发信风格
**规则**：开发信必须简洁（≤150词），直接提产品，无客套话
**适用范围**：所有海外客户开发信
```

---

### 3. 模式信号（Pattern）

**触发条件**：
- 相同指令重复3次以上
- 相同工作流成功执行3次以上
- 用户赞扬某个方法3次以上

**处理流程**：
```
检测到重复 → 记录到观察列表
↓
3次/7天内 → 标记为候选模式
↓
下次使用时确认 → 晋升到HOT (memory.md)
```

**记录位置**：
- 观察列表：`~/self-improving/patterns-watching.md`
- 确认后：`~/self-improving/memory.md`

**示例**：
```markdown
## 模式：美国市场客户发现
**工作流**：特易海关数据 → Exa背调 → LinkedIn决策人
**成功率**：85% (17/20)
**晋升日期**：2026-04-02
```

---

### 4. 失败信号（Failure）

**触发条件**：
- 工具执行失败
- 流程回退
- 用户中断任务
- 错误重试≥2次

**记录位置**：`PRACTICE-CASES.md` (实践案例库)

**记录格式**：
```markdown
## 失败案例：[简短描述]
**日期**：[日期]
**任务**：[目标]
**失败点**：[具体步骤]
**错误原因**：[分析]
**回退方案**：[实际使用的方法]
**教训**：[总结]
```

**示例**：
```markdown
## 失败案例：特易海关数据抓取失败
**日期**：2026-04-01
**任务**：查找美国传送带分销商
**失败点**：Step 1 客户发现
**错误原因**：特易网站登录超时
**回退方案**：切换到Exa搜索
**教训**：海外客户发现优先用Exa，特易作为备用
```

---

### 5. 优化信号（Optimization）

**触发词**：
- "可以更好"、"改进"
- "优化一下"、"提升"
- "下次注意"

**处理流程**：
```
识别优化建议 → 评估可行性
↓
可行 → 记录到优化队列
↓
下次执行时应用 → 验证效果
↓
有效 → 晋升为规则
```

**记录位置**：`~/self-improving/optimizations.md`

**记录格式**：
```markdown
## 优化：[主题]
**现状**：[当前做法]
**建议**：[改进方向]
**预期效果**：[量化目标]
**状态**：待验证/已验证/已否决
```

**示例**：
```markdown
## 优化：开发信响应率
**现状**：平均响应率 8%
**建议**：在开发信中加入客户公司近期动态
**预期效果**：响应率提升至 12%
**状态**：待验证
```

## 自动晋升/降级规则

### 晋升规则

| 条件 | 晋升路径 |
|------|---------|
| 模式使用3次/7天 | 观察列表 → HOT |
| 修正出现3次/7天 | corrections.md → HOT |
| 优化验证有效 | optimizations.md → HOT |

### 降级规则

| 条件 | 降级路径 |
|------|---------|
| 规则30天未使用 | HOT → WARM (projects/) |
| 规则90天未使用 | WARM → COLD (archive/) |

### 容量管理

| 层级 | 容量限制 | 超出处理 |
|------|---------|---------|
| HOT | ≤100行 | 合并相似规则 + 降级旧规则 |
| WARM | ≤200行/文件 | 按主题拆分 + 归档冷门 |
| COLD | 无限制 | 按日期归档 |

## 学习信号优先级

| 优先级 | 信号类型 | 处理时效 |
|--------|---------|---------|
| 🔴 P0 | 修正信号 | 立即记录，下次应用 |
| 🟠 P1 | 失败信号 | 任务结束后记录 |
| 🟡 P2 | 偏好信号 | 会话结束时记录 |
| 🟢 P3 | 优化信号 | 评估后记录 |
| 🔵 P4 | 模式信号 | 积累3次后处理 |

## 实际应用示例

### 场景1：工具选择错误

```
用户："不对，LinkedIn决策人应该用Exa搜索，不是linkedin工具"
AI：检测到修正信号 → 记录到corrections.md
    → 更新HOT规则：decision_maker_search → exa
    → 下次自动应用
```

### 场景2：流程失败

```
AI：特易海关数据抓取失败 → 切换到Exa
    → 检测到失败信号 → 记录到PRACTICE-CASES.md
    → 提取教训：海外客户优先Exa
    → 更新路由表优先级
```

### 场景3：工作流优化

```
用户："开发信可以更简洁一些，去掉客套话"
AI：检测到优化信号 → 记录到optimizations.md
    → 下次生成开发信时应用
    → 验证响应率提升 → 晋升为HOT规则
```

## 与Self-Improving技能的协作

本技能的`LEARNING-SIGNALS.md`与`self-improving`技能配合使用：

| 职责分工 | 本技能 | self-improving技能 |
|---------|--------|-------------------|
| **信号识别** | 获客场景特定信号 | 通用学习信号 |
| **存储位置** | 领域记忆 | 全局记忆 |
| **晋升规则** | 领域特定（3次/7天） | 通用（3次/7天） |
| **文件结构** | `~/self-improving/projects/honglong-acquisition.md` | `~/self-improving/memory.md` |

**协作流程**：
1. 本技能识别获客相关学习信号
2. 记录到领域文件 (`projects/honglong-acquisition.md`)
3. 同时通知`self-improving`技能记录到全局`corrections.md`
4. 领域规则优先级 > 全局规则
