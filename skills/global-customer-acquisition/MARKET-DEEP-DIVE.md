# 单一市场深度调研 SOP

> 🗓️ **首次实战验证**：2026-03-31，埃及工业皮带市场，找到139家海关买家 + 8家Exa补充，生成完整客户报告
>
> 📌 **适用场景**：对某个具体市场（国家 + 产品品类）进行系统性深度挖掘，产出分级客户名单 + 行业洞察报告

## 各步骤详细操作

### Step 1-2：特易搜索 + 提取（JS自动化）

参见 `teyi-customs/SKILL.md` 中的**方式4：JS自动化流程**，完整 7 步。

关键命令速查：
```bash
# 点击国家（搜索页）
agent-browser eval "Array.from(document.querySelectorAll('.country span')).find(el=>el.textContent.trim()==='埃及').closest('.country').click()"

# 提取公司列表（结果页）
agent-browser eval "[...new Map(Array.from(document.querySelectorAll('a[href*=detail]')).map(el => [el.textContent.trim(), el.href])).entries()].map(([name, href]) => name + ' | ' + href).join('\n')"
```

### Step 3：详情页背调

```bash
# 进入详情页
agent-browser open "[detail_url]"
agent-browser wait --load networkidle

# 分段读取内容（避免空白）
agent-browser eval "document.body.innerText.slice(0, 3000)"
agent-browser eval "document.body.innerText.slice(3000, 6000)"
```

提取重点字段：
- 公司全名 + 简称
- 国家/地区 + 详细地址
- 进口总次数 + 目标产品相关次数
- 总采购金额 + 最新交易日期
- 主要供应商国（判断竞争格局）
- 官网 / 联系人 / 邮箱 / 电话

### Step 4：Exa 补充

```
搜索词：conveyor belt buyer Egypt
         industrial belt supplier Egypt contact
         "[company name]" Egypt email
```

重点补充：已在特易找到但缺少联系方式的 S/A 级客户。

### Step 5-6：分析 + 分级

ICP 评分参考（**特易数据专项版**）：

| 维度 | 满分 | 计算方法 |
|------|------|----------|
| 行业匹配度 | 20 | 传送带相关采购次数 / 总次数：≥50%=20, 20-50%=10, <20%=0 |
| 采购能力 | 20 | S=$1M+(20), A=$500k+(16), B=$100k+(12), C=$50k+(8), D=$10k+(4) |
| 采购频率 | 20 | 进口次数：>100=20, 20-100=10, <20=5 |
| 客户类型 | 15 | 制造商=15, 经销商=12, 终端=8, 贸易商=5 |
| 中国采购经验 | 15 | 有中国采购记录=15, 有亚洲记录=8, 无=0 |
| 联系方式 | 10 | 直接邮箱/电话=10, 官网=5, 无=0 |

**总分 → 等级：90+(S), 70-89(A), 50-69(B), <50(C)**

## 经验总结（埃及调研 2026-03-31）

### ✅ 有效策略
1. **特易"采购商"标签** 比"全部"筛选更精准，排除了中间商和物流公司
2. **行业优先级**：水泥厂是埃及最大皮带买家群体（~40%），优先开发
3. **竞品情况**：埃及主要从印度和中国进口，需要突出质量/服务差异化
4. **Exa补充**：特易联系方式较少，Exa能补充官网邮箱/WhatsApp（特别适合中小买家）
5. **分层开发**：S级（水泥大厂）立即发开发信，A级准备中，B/C级等联系方式补充

### ⚠️ 注意事项
1. 特易数据滞后约1-3个月，"最新交易日期"是参考值
2. 部分大型集团（如 Heidelberg Materials）旗下多个子公司都是买家，算1个决策链
3. 中间商（如 ASEC 工程公司）可能帮多个工厂采购，开发价值高但决策链复杂
4. 埃及公司通常有WhatsApp，优先于邮件（本地沟通习惯）

**多市场批量扫描经验（2026-03-31）**：
- 登录后 `find first "input[type=text]" fill "关键词"` + `find role button click --name "查询"` 是最稳定的填词方式

- 多市场扫描速度：4个国家约 90 分钟（含登录/重复登录时间）

## 多市场并行扫描 SOP（2026-03-31 新增）

> 当需要一次性调研 3+ 个市场时，用这个流程代替逐市场深挖。
>
> **实战验证：** 南美+北美 4 个市场，90 分钟内完成扫描 + 综合报告

### 适用场景

- 新市场探索：快速评估多个候选市场的规模和质量
- 季度更新：批量刷新各市场的买家数据
- 竞品分析：批量查看各市场的供应商来源国

### 标准流程（4步法）

```
Step A：批量扫描（每个市场约 20 分钟）
  对每个目标国家：
  1. open "https://et.topease.net/gt/company?wlf=gt_company"
  2. wait 3000
  3. JS 点击国家
  4. find first "input[type=text]" fill "关键词"
  5. find role button click --name "查询"
  6. wait 5000
  7. 记录：总数（tab 数）+ 第一页公司列表（innerText）

Step B：横向对比分析（15分钟）
  - 整理各市场规模数据
  - 识别相似公司类型
  - 找出每个市场的 TOP3 潜力客户

Step C：重点客户背调（20分钟）
  - 对每个市场选出最高价值的 1-2 家做详情页背调
  - web_search 补充联系方式

Step D：综合报告生成（15分钟）
  - 使用多市场报告模板（见下方）
  - 包含：市场对比表 + 各市场 TOP 客户 + 竞品来源 + 优先级行动清单
```
