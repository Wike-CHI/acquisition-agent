# 联系方式验证清单

> 在发送任何开发信之前，必须完成此清单的所有检查项。

---

## ✅ 发送前验证清单

### 1. 信息来源验证（必须≥2个来源）

- [ ] 来源1：官网Contact页面（最可靠）
- [ ] 来源2：特易详情页"联系方式"（6个月内）
- [ ] 来源3：LinkedIn决策人（1年内，仅作补充）
- [ ] 来源4：易搜邮验证（决策人邮箱）

**如果只有1个来源**：❌ 禁止发送，必须找到第2个来源验证

### 2. 时效性验证

- [ ] 官网Contact页面当前有效（访问并确认）
- [ ] 特易详情页更新时间在6个月内
- [ ] LinkedIn Profile更新时间在12个月内
- [ ] 没有使用exa-search-free的结果直接发送

**如果信息超过6个月**：❌ 禁止发送

### 3. 决策人验证

- [ ] 职位是：Purchasing Manager / Procurement Director / VP
- [ ] 不是普通员工（Sales/Engineer/HR等）
- [ ] 不是2年前的旧信息（可能已调岗）
- [ ] 有公司邮箱后缀（不是Gmail/Hotmail等个人邮箱）

### 4. 多源一致性验证

- [ ] LinkedIn信息与官网信息一致
- [ ] 特易数据与LinkedIn信息一致
- [ ] 邮箱格式符合公司标准（firstname@company.com）

**如果信息不一致**：⚠️ 标记为"需人工确认"

---

## ❌ 禁止发送的情况

出现以下任何一种情况，**禁止发送**开发信：

1. ❌ 只有LinkedIn单一来源，且Profile更新时间超过12个月
2. ❌ 使用exa-search-free找到的邮箱，未经过官网/特易验证
3. ❌ 联系人职位是普通员工（非采购决策人）
4. ❌ 找不到信息发布时间（无法验证时效性）
5. ❌ LinkedIn显示的职位与特易/官网不一致
6. ❌ 邮箱是个人邮箱（Gmail/Hotmail/Yahoo等）

---

## 🚨 越南客户案例教训回顾

**错误做法**：
```
1. AI使用exa-search-free搜索
2. 找到LinkedIn上一个"采购经理"
3. 直接发送邮件
4. 结果：客户回复已调岗2年
```

**正确做法**：
```
1. AI使用exa-search-free搜索（仅作线索）
2. 访问官网Contact页面（验证公司存在）
3. 使用特易详情页验证邮箱（验证时效性）
4. 使用易搜邮查找最新决策人邮箱
5. 确认信息在6个月内
6. 发送邮件（多源验证通过）
```

---

## ✅ 安全发送流程

```
步骤1: 搜索（exa/LinkedIn/特易）→ 获取线索
   ↓
步骤2: 官网验证 → 访问Contact页面
   ↓
步骤3: 特易验证 → 查看详情页"联系方式"（6个月内）
   ↓
步骤4: 易搜邮补充 → 查找决策人邮箱
   ↓
步骤5: 多源对比 → 确认≥2个来源一致
   ↓
步骤6: 决策 → 全部通过 → 发送 / 任一失败 → 放弃
```
