# delivery-queue — 消息分段发送

模拟真人发送节奏，避免被标记为垃圾信息。

---

## 🎯 使用场景

| 场景 | 说明 |
|------|------|
| **长产品介绍** | 分成3-5条消息发送 |
| **跟进消息** | 定时发送（如客户本地时间9点） |
| **Drip Campaign** | 多天分批发送 |
| **避免垃圾标记** | 控制发送频率 |

---

## 📋 命令

| 匇令 | 功能 |
|------|------|
| `deliver:schedule` | 安排消息定时发送 |
| `deliver:list` | 查看待发送消息 |
| `deliver:cancel` | 取消待发送消息 |
| `deliver:flush` | 立即发送所有消息 |

---

## ⚙ 配置

```yaml
default_delay_ms: 3000        # 消息间默认延迟（3秒）
max_segments: 10              # 每次最多分段数
timezone: "Asia/Shanghai"    # 发送者时区
quiet_hours:                  # 静默时段（不发送）
  start: "22:00"
  end: "07:00"
```

## 🔧 宂后逻辑

### 分段规则

| 消息长度 | 分段数 | 每段延迟 |
|----------|--------|----------|
| < 100字 | 1段 | 无 |
| 100-300字 | 2段 | 3秒 |
| 300-500字 | 3段 | 5秒 |
| > 500字 | 4-5段 | 8秒 |

### 静默时段

- **22:00 - 07:00** 不发送消息
- 周末降低频率（可选）
- 节假日暂停发送（可选）

### 重试机制

- 发送失败自动重试（最多3次）
- 重试间隔： 30秒 → 60秒 → 120秒
- 3次失败后标记为发送失败

---

## ⚠️ 注意事项

1. **不要过度分段** - 保持消息完整性
2. **尊重时区** - 在客户工作时间发送
3. **避免垃圾模式** - 同一客户每小时最多5条消息
4. **自然语言** - 分段后保持对话流畅
