# 获客工作流技能

端到端获客流程定义，包括技能调度、质量检查、结果输出。

## 流程总览

```
Phase 1: 搜索 (10分钟)
    │
    ▼
Phase 2: 背调 (30分钟)
    │
    ▼
Phase 3: 筛选 (5分钟)
    │
    ▼
Phase 4: 触达 (10分钟)
```

## 阶段详解

### Phase 1: 搜索
**目标**: 发现15-30家潜在客户
**技能选择**:
| 场景 | 调用技能 |
|------|----------|
| LinkedIn搜索 | linkedin |
| 海关数据 | scrapling + 浏览器 |
| Google搜索 | multi-search-engine |
| Facebook挖掘 | scrapling |

**质量检查点**:
- [ ] 搜索关键词是否准确？
- [ ] 结果是否相关？
- [ ] 是否有联系方式？

**输出**: 公司列表 (JSON格式)

### Phase 2: 背调
**目标**: 评估客户质量
**技能选择**:
| 场景 | 调用技能 |
|------|----------|
| 快速背调 | company-research |
| 深度背调 | in-depth-research |
| 市场分析 | market-research |
| 竞品识别 | honglong-products |

**质量检查点**:
- [ ] 评分是否合理？
- [ ] 是否识别出竞争对手？
- [ ] 关键信息是否完整？

**输出**: 评分报告 (A/B/C/D级)

### Phase 3: 筛选
**目标**: 确定高价值客户
**筛选条件**:
- 评分 ≥ 75分
- 非竞争对手
- 有联系方式

**输出**: 高价值客户列表 (2-5家)

### Phase 4: 触达
**目标**: 发送带产品资料的开发信
**技能选择**:
| 场景 | 调用技能 |
|------|----------|
| NAS挂载 | nas-file-reader |
| 获取产品资料 | nas-file-reader + honglong-products |
| 生成开发信 | email-marketing |
| 配置邮箱 | email-sender |
| 发送邮件 | email-sender |

**前置检查**:
- [ ] NAS是否已挂载？
- [ ] 邮箱是否已配置？
- [ ] 产品图片是否可用？

**详细步骤**:
1. **挂载NAS** - 获取产品图片/视频
2. **准备资料** - 选择合适的产品图片
3. **生成开发信** - 个性化+产品链接
4. **配置邮箱** - 首次需要运行setup脚本
5. **发送邮件** - 遵守频率限制

**质量检查点**:
- [ ] 开发信是否个性化？
- [ ] 是否包含产品资料？
- [ ] 发送是否成功？
- [ ] 是否设置跟进提醒？

**输出**: 发送记录 + 跟进计划 + 产品资料链接

## 快速命令

### 搜索命令
```
帮我搜索 [数量] 个 [地区] 的 [行业] 客户
```

### 背调命令
```
背调这些公司： [公司列表]
```

### 筛选命令
```
筛选高价值客户，标准：评分≥[分数] 且非竞争对手
```

### 触达命令
```
给这些客户发开发信: [公司列表]
```
