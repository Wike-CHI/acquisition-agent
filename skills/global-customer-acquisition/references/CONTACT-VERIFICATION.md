# 联系方式验证指南

> 防止使用旧信息、错误信息发送开发信，避免发错人。
>
> 核心原则：多源验证 + 时效性检查 + 决策人确认

---

## 🚨 为什么需要验证？

### 越南客户案例教训

**错误做法**：
```
1. AI使用exa-search-free搜索
2. 找到LinkedIn上一个"采购经理"的邮箱
3. 没有验证信息时效性
4. 直接发送邮件
```

**后果**：
- 客户回复：2年前已调岗，不是决策人
- 浪费开发信
- 留下不好印象

**正确做法**：
```
1. AI使用exa-search-free搜索（仅作线索）
2. 验证信息时效性（6个月内）
3. 多源验证（LinkedIn + 官网 + 特易）
4. 确认决策人职位
5. 发送邮件
```

---

## ⚠️ 禁止使用的联系方式

### 绝对禁止

| 情况 | 原因 | 后果 |
|------|------|------|
| **超过6个月的LinkedIn信息** | 职位可能变更 | 发错人 |
| **发布日期未知的公网邮箱** | 无法验证时效性 | 可能是旧信息 |
| **仅单一来源** | 无交叉验证 | 可靠性低 |
| **LinkedIn普通员工** | 不是决策人 | 浪费时间 |
| **个人邮箱（Gmail等）** | 不专业 | 到达率低 |

### 禁用示例

❌ **错误1**：使用2年前的LinkedIn信息
```
LinkedIn Profile: John Smith (Purchasing Manager)
最后更新：2023年6月（超过6个月）
→ 禁止使用
```

❌ **错误2**：使用发布日期未知的公网邮箱
```
公网搜索找到：john@company.com
发布日期：未知
→ 禁止使用
```

❌ **错误3**：仅凭LinkedIn单一来源
```
LinkedIn找到：mary@company.com
没有官网/特易验证
→ 禁止使用
```

## 🔍 验证流程（Phase 1.5 必须）

### 完整验证流程

```
找到联系方式 → 验证时效性 → 多源确认 → 决定使用/放弃
```

### 步骤详解

#### 步骤1：时效性检查

**检查项**：
- [ ] LinkedIn Profile最后更新时间（12个月内）
- [ ] 特易详情页数据更新时间（6个月内）
- [ ] 官网Contact页面当前有效
- [ ] exa-search-free结果发布时间（未知）

**通过标准**：
- ✅ S级/A级：当前有效或6个月内
- ✅ B级：12个月内
- ❌ 任何超过时限的：禁止使用

#### 步骤2：多源验证

**验证方法**：
1. **LinkedIn**：查看Profile，确认职位
2. **官网**：访问Contact页面，查找邮箱
3. **特易**：查看详情页"联系方式"

**一致性检查**：
- LinkedIn姓名 ≈ 官网姓名 ≈ 特易姓名
- LinkedIn职位 ≈ 官网职位 ≈ 特易职位
- 邮箱后缀 = 公司官网域名

**通过标准**：
- ✅ ≥2个来源一致：使用
- ⚠️ 只有1个来源：需人工确认
- ❌ 来源不一致：放弃

#### 步骤3：决策人确认

**确认项**：
- [ ] 职位是：Purchasing Manager
- [ ] 职位是：Procurement Director
- [ ] 职位是：VP / Director
- [ ] 职位是：Owner / CEO

**通过标准**：
- ✅ 职位是决策人：使用
- ❌ 职位是普通员工（Sales/Engineer/HR）：禁止使用

#### 步骤4：决定使用/放弃

| 情况 | 决定 |
|------|------|
| 时效性通过 + 多源验证通过 + 决策人确认 | ✅ 使用 |
| 时效性不通过 | ❌ 放弃 |
| 多源验证不通过 | ❌ 放弃 |
| 决策人不确认 | ❌ 放弃或标记"需人工确认" |

## 🚨 exa-search-free 使用警告

> **重要**：`exa-search-free` 和 `mcporter call 'exa.web_search_exa'` 可能返回**旧信息**。

### 使用规则

**禁止**：
- ❌ 直接使用 `exa-search-free` 找到的邮箱发送开发信
- ❌ 相信 `exa-search-free` 结果的时效性

**必须**：
- ✅ 用 `exa-search-free` 结果作为**线索**，再进行验证：
  - 访问官网Contact页面
  - 使用特易详情页"联系方式"验证
  - 使用易搜邮工具查找最新邮箱
- ✅ 如果找不到时效性验证，**放弃该客户**

### 使用流程

```
exa-search-free搜索 → 获取线索（不直接使用）
   ↓
官网验证 → 访问Contact页面
   ↓
特易验证 → 查看详情页"联系方式"
   ↓
易搜邮补充 → 查找最新决策人邮箱
   ↓
多源对比 → 确认信息一致性
   ↓
决定使用/放弃
```

## ⏰ 时效性验证标准

### 信息来源时效要求

| 信息来源 | 时效要求 | 验证方法 |
|---------|---------|---------|
| **LinkedIn Profile** | ≤ 12个月 | 查看"Activity"或"About"最后更新时间 |
| **特易详情页** | ≤ 6个月 | 查看数据更新日期 |
| **官网Contact页面** | 当前有效 | 访问官网确认页面存在 |
| **exa-search-free结果** | ❌ 禁止使用 | 仅作为线索，必须二次验证 |

### 验证方法

#### LinkedIn Profile时效性

1. 访问LinkedIn Profile
2. 查看"Activity"部分
3. 找到最后更新时间
4. 确认是否在12个月内

#### 特易详情页时效性

1. 访问特易详情页
2. 查看"商业数据"部分
3. 找到数据更新日期
4. 确认是否在6个月内

#### 官网Contact页面时效性

1. 访问官网
2. 找到Contact页面
3. 确认页面存在且可访问
4. 验证邮箱后缀是否正确

## 📊 验证检查清单

在发送开发信之前，必须完成以下检查：

### 时效性检查

- [ ] 信息发布时间在6个月内（官网/特易）
- [ ] 信息发布时间在12个月内（LinkedIn）
- [ ] 没有使用exa-search-free结果直接发送

### 多源验证

- [ ] 有≥2个来源验证
- [ ] LinkedIn信息与官网信息一致
- [ ] 特易数据与LinkedIn信息一致
- [ ] 邮箱格式符合公司标准

### 决策人确认

- [ ] 职位是Purchasing/Procurement/VP/Director
- [ ] 不是普通员工（Sales/Engineer/HR）
- [ ] 不是2年前的旧信息
- [ ] 有公司邮箱后缀（不是Gmail等）

### 最终决定

- [ ] 全部通过 → 使用
- [ ] 任一失败 → 放弃或标记"需人工确认"

## 🔧 工具使用指南

### 工具1：特易详情页

**用途**：验证公司基本信息和联系方式

**操作步骤**：
1. 搜索公司名
2. 点击公司名进入详情页
3. 查看"商业数据" → "联系方式"
4. 确认邮箱和更新时间

### 工具2：易搜邮

**用途**：查找决策人邮箱

**操作步骤**：
1. 特易平台 → 易搜邮
2. 输入公司名/网站
3. 查找决策人邮箱
4. 确认职位和邮箱

### 工具3：LinkedIn

**用途**：确认决策人信息

**操作步骤**：
1. 搜索公司名
2. 查看员工列表
3. 找到决策人（Purchasing Manager）
4. 查看Contact Info

### 工具4：官网Contact页面

**用途**：验证公司存在和联系方式

**操作步骤**：
1. 访问官网
2. 找到Contact页面
3. 查看联系表单或邮箱
4. 确认公司存在

## ❓ 常见问题

### Q1：如果所有验证都失败怎么办？

**A**：放弃该客户，或标记为"需人工确认"。
- 不要冒险发错人
- 节省时间在更可靠的客户上

### Q2：如果特易和LinkedIn信息不一致怎么办？

**A**：相信特易（6个月内数据），LinkedIn可能过期。
- 特易数据更新：6个月内
- LinkedIn Profile更新：可能很久没更新
- 优先使用特易数据

### Q3：如果找不到决策人邮箱怎么办？

**A**：使用官网"Contact Us"表单。
- 表单直接到达公司
- 比通用邮箱（info@）更可靠
- 在表单中说明想要联系采购部门

### Q4：exa-search-free结果完全不能用吗？

**A**：可以作为线索，但必须二次验证。
- 用exa找到公司名
- 再去官网/特易验证
- 不能直接用exa找到的邮箱
