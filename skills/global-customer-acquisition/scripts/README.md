# 红龙获客系统 - 一键初始化指南

> 🎯 目标：将首次使用时间从 30-60 分钟降低到 **5 分钟**

---

## 快速开始

### Windows PowerShell

```powershell
# 进入技能目录
cd $env:USERPROFILE\.workbuddy\skills\global-customer-acquisition

# 运行初始化脚本
.\scripts\init.ps1

# 如果遇到执行策略错误，先运行：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## 常见问题

### Q1: 提示"无法加载脚本，因为在此系统上禁止运行脚本"

**原因**：PowerShell 执行策略限制

**解决**：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

---

### Q2: 凭据配置需要在哪里填写？

**推荐方式**：使用专门的配置命令

```bash
# 配置邮箱
配置邮箱

# 配置 Exa API
配置 Exa

# 配置 DeepSeek API
配置 DeepSeek
```

这些命令会使用凭据管理技能安全存储密钥。

---

### Q3: 哪些凭据是必需的？

| 凭据 | 必需 | 用途 |
|------|------|------|
| **SMTP** | ✅ 必需 | 发送开发信 |
| Exa API | ⚠️ 可选 | 客户发现（默认使用免费版） |
| DeepSeek API | ⚠️ 可选 | 开发信生成（可使用其他模型） |

---

### Q4: 技能依赖缺失怎么办？

脚本会自动标记缺失的技能：

```
⚠️  缺失技能:
   - chroma-memory 【必需】
   - supermemory 【必需】
   - linkedin 【可选】
```

**安装必需技能**：
1. 在 WorkBuddy 左侧点击"Skills"
2. 搜索技能名称
3. 点击安装

## 参数说明

```powershell
.\scripts\init.ps1 [参数]
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-SkipChecks` | 跳过依赖检查（不推荐） | `$false` |
| `-Interactive` | 启用交互式配置向导 | `$true` |

**示例**：
```powershell
# 仅检查，不交互
.\scripts\init.ps1 -Interactive:$false

# 完整流程
.\scripts\init.ps1 -Interactive:$true
```

## 技术细节

### 检测逻辑

1. **业务员信息检测**
   - 读取 `context/user.md`
   - 检查是否包含占位符（"请填写"）
   - 验证邮箱格式（警告，不阻止）

2. **凭据配置检测**
   - 读取 `TOOLS.md`
   - 检查是否包含占位符（`{{...}}`）
   - 区分必需/可选凭据

3. **技能依赖检测**
   - 扫描 `~/.workbuddy/skills/`
   - 检查必需技能是否安装
   - 列出可选技能缺失项

### 文件修改

脚本**仅修改** `context/user.md`，其他文件只读取不修改。

### 安全性

- ✅ 不读取外部 API 密钥
- ✅ 不发送任何数据到外部
- ✅ 仅操作本地配置文件
- ✅ 凭据推荐使用专业技能配置
