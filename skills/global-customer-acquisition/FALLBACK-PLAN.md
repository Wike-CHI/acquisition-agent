> 官方推荐：每步之间要有验证——上一步成功了再走下一步。
> 特别提醒要写好回退方案：如果第三步失败了，前两步怎么处理。

## 完整流程 + 回退方案

```
┌─────────────────────────────────────────────────────────────┐
│  获客流程 + 验证 + 回退                                    │
├─────────────────────────────────────────────────────────────┤
│  Step 1: 客户发现                                            │
│     ├── ✅ 成功 → 保存客户数据 → 继续                         │
│     └── ❌ 失败 → 回退方案A                                  │
│                                                             │
│  Step 2: 客户查重                                            │
│     ├── ✅ 无重复 → 继续                                     │
│     ├── ⚠️ 有重复 → 合并记录 → 继续                         │
│     └── ❌ 错误 → 回退方案B                                  │
│                                                             │
│  Step 3: 企业背调                                            │
│     ├── ✅ 成功 → ICP评分 → 继续                             │
│     ├── ⚠️ 信息不足 → 标记为"待补充" → 继续                 │
│     └── ❌ 失败 → 回退方案C                                  │
│                                                             │
│  Step 4: LinkedIn 决策人                                     │
│     ├── ✅ 找到 → 生成决策人报告 → 继续                      │
│     ├── ⚠️ 未找到 → 标记为"需人工查找" → 继续               │
│     └── ❌ 渠道不可用 → 回退方案D                            │
│                                                             │
│  Step 5: 开发信生成                                          │
│     ├── ✅ 质量合格 → 保存草稿 → 继续                        │

│                                                             │
│  Step 6: 邮件发送                                            │
│     ├── ✅ 成功 → 记录发送 → 进入跟进流程                    │
│     ├── ⚠️ 邮箱无效 → 尝试其他邮箱                          │
│     └── ❌ 发送失败 → 回退方案E                              │
└─────────────────────────────────────────────────────────────┘
```

## 各步骤回退方案

### 回退方案A: 客户发现失败

**失败原因**:
- 海关数据搜索无结果
- LinkedIn搜索无结果
- Exa搜索无结果

**回退操作**:
```
1. 记录失败原因
2. 尝试其他关键词
3. 切换其他渠道
4. 如果所有渠道都失败：
   - 建议用户更换搜索条件
   - 保存失败记录供后续分析
```

**数据保留**:
- ✅ 保留：搜索关键词、尝试的渠道
- ❌ 不保留：无用的临时数据

**示例**:
```
用户: 搜索美国传送带制造商

AI:
🔍 尝试渠道1: 特易海关数据...
❌ 无结果

🔍 尝试渠道2: Exa搜索...
❌ 无结果

🔍 尝试渠道3: LinkedIn搜索...
❌ 无结果

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⚠️ 所有渠道都未找到结果

建议：
1. 更换关键词（如 "belt conveyor" 而非 "conveyor belt"）
2. 扩大地区范围（如 "北美" 而非 "美国"）
3. 减少筛选条件

已记录本次搜索失败，供后续分析。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 回退方案B: 客户查重错误

**失败原因**:
- CRM系统无法连接
- 查重脚本错误
- 数据库损坏

**回退操作**:
```
1. 尝试本地缓存查重
2. 如果缓存也不可用：
   - 跳过查重，标记为"未查重"
   - 继续后续流程
   - 人工确认时提醒用户查重
```

**数据保留**:
- ✅ 保留：已获取的客户数据
- ✅ 标记：添加"未查重"标签

**示例**:
```
AI:
🔍 正在查重...
❌ CRM系统连接失败

⚠️ 查重失败，跳过查重步骤

✅ 已标记为"未查重"
💡 后续需要人工确认是否重复

继续企业背调...
```

### 回退方案C: 企业背调失败

**失败原因**:

```
2. 如果所有渠道都失败：
   - 标记为"信息不足"
   - 基于有限信息进行初步评分
```

**数据保留**:
- ✅ 保留：已获取的有限信息
- ✅ 标记：添加"信息不足"标签
- ✅ 评分：基于有限信息的初步评分

```
- 行业匹配度：7/10（推测）
```

### 回退方案D: LinkedIn渠道不可用

**回退操作**:

- ✅ 保留：已获取的客户数据
- ✅ 标记：添加"需人工查找决策人"标签

**示例**:
```
AI:
```

### 回退方案E: 邮件发送失败

**失败原因**:
- SMTP 连接失败
- 邮箱无效（退信）
- 发送频率超限

**回退操作**:
```
1. 记录失败原因
2. 尝试其他邮箱（如果有）

   - 标记为"发送失败"
   - 加入重试队列（1小时后重试）
   - 通知用户
```

**数据保留**:
- ✅ 保留：开发信草稿
- ✅ 保留：客户信息
- ✅ 记录：失败原因和时间
- ✅ 加入：重试队列

## 步骤间验证

### 验证点1: 客户发现后

**验证内容**:
```
✅ 至少找到1个客户？
✅ 客户信息完整（公司名+国家）？
✅ 数据格式正确？
```

**失败处理**:
```
❌ 未找到客户 → 回退方案A
❌ 信息不完整 → 尝试补充，失败则回退方案A
❌ 格式错误 → 修复格式，继续
```

---

### 验证点2: 企业背调后

**验证内容**:
```
✅ 至少获取3个维度信息？
✅ ICP评分计算完成？
✅ 客户等级确定？
```

**失败处理**:
```
❌ 信息不足 → 标记"信息不足"，继续
❌ 评分未完成 → 基于有限信息评分
❌ 等级未确定 → 默认C级
```

---

### 验证点3: 开发信生成后

**验证内容**:
```
✅ 质量评分 >= 7.0？
✅ 无AI感？
✅ 个性化充分？
```

**失败处理**:
```
❌ 评分 < 7.0 → 自动修改（最多3轮）
❌ 有AI感 → 去AI化处理
❌ 个性化不足 → 补充个性化元素
```

## 重试机制

### 自动重试队列

```python
class RetryQueue:
    """失败任务自动重试队列"""
    
    def __init__(self):
        self.queue = []
    
    def add(self, task, retry_after_seconds):
        """添加失败任务到重试队列"""
        self.queue.append({
            "task": task,
            "retry_at": time.time() + retry_after_seconds,
            "retry_count": 0
        })
    
    def process(self):
        """处理重试队列（每小时运行一次）"""
        now = time.time()
        for item in self.queue:
            if item["retry_at"] <= now and item["retry_count"] < 3:
                # 重试任务
                result = execute(item["task"])
                if result.success:
                    self.queue.remove(item)
                else:
                    # 失败，下次重试时间加倍
                    item["retry_at"] = now + (3600 * (2 ** item["retry_count"]))
                    item["retry_count"] += 1
```

### 重试时间表

| 重试次数 | 重试间隔 | 说明 |
|----------|----------|------|
| 第1次 | 1小时 | 快速重试 |
| 第2次 | 2小时 | 中等间隔 |
| 第3次 | 4小时 | 最后一次 |
| 放弃 | - | 通知用户 |
